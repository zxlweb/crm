## 数据范围（按角色）



| 角色 | 线索/客户/商机 | 仪表盘团队排行 |

|------|----------------|----------------|

| **租户管理员** | 全租户 | **按部门**汇总本月赢单 |

| **销售经理 / 客户经理** | 仅本事业部成员负责的数据 | **按成员**（本部门内） |

| **销售代表 / 只读** | 仅本事业部数据 | 不展示排行区块 |



部门归属写在 **设置 → 成员** 的「部门」列（来自 `user_tenants.department`）。**非租户管理员**只要配置了部门，线索/客户/商机/仪表盘 KPI 均 **仅本事业部**，无法看到其它事业部数据。

## 建议演示路径（约 8 分钟）



1. **系统设置 → 成员** — 查看潘总、各部门负责人及「部门」列

2. **仪表盘（ceo@xiaoxi.com）** — 配额 ¥2,800 万；**部门业绩排行**（灵狐、神龙、麒麟等）

3. **商机 / Deals** — 管道：灵狐中台谈判、政务云二期、信创云战略单

4. **线索** — 打开「⭐ 华东数智 · 数据中台年度」→ 情绪旅程 / 跟进时间线

5. **客户 / 联系人** — 按标签筛选「神龙云计算」「马来西亚」

6. **`linghu@xiaoxi.com`（李婷 · 销售经理）** — 仅见灵狐数据范围数据；团队排行为本部门成员

7. **`moye@xiaoxi.com`（王磊 · 销售代表）** — 仅本人线索/商机，无团队排行
