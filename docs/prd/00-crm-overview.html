CRM 系统 MVP PRD

产品名称：EnterpriseFlow CRM（企业流 CRM）
版本：MVP v0.1
日期：2026-05-21
目标用户：中小企业销售团队、客服团队

1. 背景与业务目标

随着企业数字化转型，中小企业需要一套轻量、易用、安全的多租户 CRM 系统，帮助他们更好地管理客户、销售流程和团队协作。

核心业务目标（MVP 阶段）：

  • 实现客户全生命周期管理
  • 提升销售团队效率和成交率
  • 支持多租户 SaaS 模式，快速服务多个企业客户
  • 建立完善的权限与数据隔离体系
  • 支持中英文双语

2. 用户角色 (Personas)

角色 描述 核心诉求
Tenant Admin（管理员） 企业负责人或 IT 管理员 成员管理、权限分配、租户配置
Sales Manager（销售经理） 销售团队主管 查看团队业绩、分配线索、报表
Sales（销售人员） 普通销售 跟进线索、维护客户、创建商机
Viewer（只读用户） 部分查看权限用户 查看相关数据

3. MVP 功能范围（核心模块）

Phase 1 - MVP 核心功能（建议第一个迭代完成）

1. 认证与多租户基础

  • 用户注册 / 登录 / 忘记密码
  • 多租户支持（邀请成员、切换租户）
  • Super Admin 后台（跨租户管理）
  • JWT + Refresh Token 认证

2. RBAC 权限系统

  • 角色管理（内置 + 自定义角色）
  • 权限分配（资源:操作 模式）
  • 按钮/菜单/页面级权限控制
  • 数据权限（本人可见 / 部门可见 / 全部可见）

3. 客户管理 (Contacts & Companies)

  • 公司（Accounts）管理
  • 联系人（Contacts）管理
  • 关联关系（一个公司多个联系人）
  • 基本信息 + 自定义字段

4. 线索管理 (Leads)

  • 线索录入、导入（Excel）
  • 线索分配、跟进记录
  • 线索状态流转（New → Contacted → Qualified → etc.）
  • 线索转商机

5. 商机管理 (Deals)

  • 商机创建与阶段管理（Pipeline）
  • 金额、预计成交日期、概率
  • 简单销售阶段看板（Kanban）

6. 仪表盘 (Dashboard)

  • 个人/团队关键指标（本月成交、线索数量、转化率）
  • 销售漏斗图
  • 待跟进事项

7. 系统设置

  • 租户基本信息配置
  • 自定义字段管理
  • i18n 语言切换（中 / 英）
  • 审计日志查看（管理员）

后续迭代（Phase 2+，MVP 之后）

  • 任务 / Activity 管理
  • 合同管理
  • 报表与高级分析
  • 邮件集成 / 通知
  • 移动端适配
  • API 开放平台

4. 非功能需求

  • 多租户数据隔离：严格隔离
  • 性能：单租户 ≤ 200 用户时，页面加载 < 1.5s
  • 安全：JWT 安全、密码加密、操作审计、SQL 注入防护
  • 国际化：完整中英文支持，后续可扩展
  • 技术约束：Nuxt 3 + Go + PostgreSQL
  • 可扩展性：支持后续增加模块

5. 用户故事示例（部分）

线索管理

  • 作为销售，我可以新建线索，填写基本信息和跟进记录
  • 作为销售经理，我可以把线索分配给团队成员
  • 作为管理员，我可以查看所有租户的线索统计

权限

  • 作为管理员，我可以创建角色并分配权限
  • 作为销售，我只能看到自己负责的线索和商机（默认）

6. 优先级 (MoSCoW)

  • Must Have：认证 + 多租户 + RBAC + Contacts + Leads + Deals + Dashboard
  • Should Have：自定义字段、简单看板、导入导出
  • Could Have：活动记录、邮件通知
  • Won't Have (MVP)：AI 智能分析、复杂报表、移动 App

7. 成功指标 (OKRs)

  • Objective：快速验证产品市场可用性
  • Key Results：
    • 完成 MVP 开发并成功部署
    • 支持至少 3 个租户同时使用且数据隔离正常
    • 系统核心页面响应时间 < 2s
    • 权限控制无明显越权漏洞
